# Failure History

> What has been tried and did not work. Agents read this before attempting anything non-trivial. Repeating a documented failure is a process error.

---

## Purpose

Record what has been tried and failed so agents do not repeat the same mistakes. This is institutional memory. A failure documented here is a failure that cannot happen twice in the same way.

The Tester is the primary writer. Any agent that encounters a failure while operating may add an entry.

## Entry Protocol

When documenting a failure:
- **What Was Tried**: be specific enough that another agent can recognize the same situation
- **Why It Failed**: root cause, not symptom
- **What Was Learned**: the actionable insight — what to do differently next time

Vague entries ("it didn't work") are not useful. Specific entries ("used model family X to evaluate output from model family X — produced self-congratulatory scores") are.
